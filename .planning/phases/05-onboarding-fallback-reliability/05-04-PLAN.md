```
Body: { inviteUrl: string }
Success: 200 { ok: true, serverPublicKey: string }
Error: 409 (connected), 400 (bad input), 500 (relay error)
```

"Not paired" detection:
- Use daemon status state: if daemon responds but has no peer configured, status.state will be "idle"
- A simpler heuristic: show onboarding if daemon IPC returns 200 on /health but /status shows state==="idle" with no prior connection history
- OR: add a dedicated flag in DaemonStatus (e.g., isPaired: boolean) — add this to @homelan/shared DaemonStatus type and daemon.ts
- SIMPLEST approach for v1: Show onboarding only when user explicitly triggers it via a "Pair new device" button in the dashboard. This avoids over-engineering the "first time" detection. The button opens the wizard overlay.

Decision (Claude's discretion): Implement as an overlay/modal triggered by a "Pair Device" button in the header. This is simpler than auto-detection and works reliably on both first run and subsequent uses.

Styling: Follow existing Tailwind + shadcn/ui patterns (bg-gray-950, text-white, rounded-xl, etc.)
