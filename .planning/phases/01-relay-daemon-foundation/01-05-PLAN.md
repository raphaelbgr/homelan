  <how-to-verify>
    **Test 1: Relay server starts and validates config**
    ```bash
    cd packages/relay
    # Should fail fast with clear error:
    node dist/index.js
    # Expected: error message mentioning missing RELAY_SECRET, exits non-zero

    # Should start successfully:
    RELAY_SECRET=test-secret RELAY_STORAGE=memory node dist/index.js
    # Expected: "[relay] Listening on :3000"
    ```

    **Test 2: Relay accepts peer registration**
    ```bash
    # In another terminal (relay still running):
    curl -k -X POST https://localhost:3000/register \
      -H "Content-Type: application/json" \
      -d '{"publicKey":"dGVzdGtleXRlc3RrZXl0ZXN0a2V5dGVzdGtleXQ=","endpoint":"1.2.3.4:51820","timestampMs":1234567890}'
    # Expected: {"ok":true,"ttlSeconds":300}

    curl -k "https://localhost:3000/lookup/dGVzdGtleXRlc3RrZXl0ZXN0a2V5dGVzdGtleXQ="
    # Expected: JSON with publicKey, endpoint, timestampMs
    ```

    **Test 3: Daemon starts and generates keys**
    ```bash
    cd packages/daemon
    node dist/index.js
    # Expected:
    # [daemon] No key found. Generating new WireGuard keypair...
    # [daemon] Public key: <44-char base64>
    # [daemon] IPC server listening on 127.0.0.1:30001
    ```

    **Test 4: Daemon IPC responds correctly**
    ```bash
    # In another terminal (daemon still running):
    curl http://127.0.0.1:30001/health
    # Expected: {"ok":true,"uptimeMs":<number>}

    curl http://127.0.0.1:30001/status
    # Expected: JSON with state:"idle", mode:null, latencyMs:null, throughputBytesPerSec:null, hostInfo:null, connectedPeers:[], lanDevices:[], uptimeMs:<number>

    curl http://127.0.0.1:30001/devices
    # Expected: {"devices":[]}

    curl -X POST http://127.0.0.1:30001/connect -H "Content-Type: application/json" -d '{"mode":"full-gateway"}'
    # Expected: {"error":"Not implemented...","code":"NOT_IMPLEMENTED"} with status 501
    ```

    **Test 5: SSE stream**
    ```bash
    curl -N http://127.0.0.1:30001/events
    # Expected: Hangs open, first line is:
    # data: {"type":"state_changed","timestampMs":<n>,"data":{"state":"idle"}}
    ```

    **Test 6: Daemon restart preserves keys**
    ```bash
    # Stop daemon (Ctrl+C), restart it
    node dist/index.js
    # Expected: No "Generating new WireGuard keypair..." message
    # Same public key as first run
    ```
